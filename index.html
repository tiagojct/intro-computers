Uma janela de texto. Escreves, carregas Enter, a máquina responde.

terminal — rosa

Aquela linha tem quatro partes. Cada cor é uma:

rosa — quem és (o utilizador)
~/estudo — onde estás (a pasta)
ls — a ordem que escreves (aqui, "listar")
por baixo — a resposta da máquina

A forma do pedido

Só um impresso de requisição

Os computadores precisam do pedido num formato fixo, com campos. Isso é uma API.

texto solto
difícil de processar
pedido estruturado (API)
exame:hemograma
doente:0412
urgência:normal
campos claros, sempre iguais
O impresso que o laboratório aceita: campos, sem ambiguidade.
A tua credencial

A chave de API

Uma chave que te identifica no serviço. É pessoal.

a tua chave pessoal · • • • • • •
Identifica-te ao serviço: ele sabe que o pedido é teu.
Nunca a partilhes nem a escrevas onde outros a vejam.
Pode ter custos: cada pedido ao serviço pode ser pago.
Como o teu login de prescritor: é tua, é a tua responsabilidade.

A IA pode enganar-se

Útil, mas não é fonte de verdade. Pode errar com toda a confiança.

Não conhece o teu doente nem o contexto clínico.

Quando os ficheiros têm dados de doentes

Não é uma conveniência

Enviar dados identificáveis para fora é tratamento de dados (RGPD). Três perguntas, primeiro:

Há um contrato com quem trata os dados?
Onde (em que país) ficam guardados os dados?
Há base legal e consentimento para este uso?
Sem resposta clara, não envies para a cloud. Na dúvida, fala com o DPO.
Para dados de doentes: a opção segura

Manter tudo na máquina

Para dados identificáveis, a escolha segura é uma IA que corre na tua própria máquina (com o Ollama, da lista verificada). Nada sai.
