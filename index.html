A mesma estrutura. Muda só a forma de a escrever.

macOS /Users/rosa/Hospital/Cardiologia/Doente-0412/exame.pdf

A janela onde se escreve

O terminal

Uma janela de texto. Escreves, carregas Enter, a máquina responde.

terminal — rosa

Aquela linha tem quatro partes. Cada cor é uma:

rosa — quem és (o utilizador)
~/estudo — onde estás (a pasta)
ls — a ordem que escreves (aqui, "listar")
por baixo — a resposta da máquina

Tudo o que está fora dessa pasta fica de fora. É o teu espaço de trabalho.

Quando os ficheiros têm dados de doentes

Não é uma conveniência

Enviar dados identificáveis é tratamento de dados (RGPD). Três perguntas:

Há um contrato com quem trata os dados?
Onde ficam guardados?
Há base legal e consentimento?
Sem resposta clara às três: não enviar para a cloud usar modelo local falar com o DPO
Antes de usar, despir os dados

Pseudonimização

Tirar o que identifica o doente antes de o ficheiro ser usado.

NomeJoão M. Silva█████████
Nº utente123456789█████████
Dados clínicostensão, idade, exame
Reduz o risco — mas confirma sempre a base legal. Não é um passe livre.
Para dados de doentes: a opção segura

Manter tudo na máquina

Para dados identificáveis, a escolha segura é uma IA que corre na tua própria máquina (com o Ollama, da lista verificada). Nada sai.

nada atravessa

Privado por construção. A cloud só sem dados de doentes (ou com contratos e consentimento).
